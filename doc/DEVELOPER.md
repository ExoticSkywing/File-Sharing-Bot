# 🌱 小芽空投机 开发文档

> 面向开发者和运维人员的详细技术说明

---

## 📌 角色说明

| 角色 | 做什么 |
|------|-------|
| **管理员** | 打包资源、生成分享链接 |
| **用户** | 点击链接 → 发送口令 → 领取资源 |

---

## 🔧 管理员操作

### 第一步：开启存储会话

在 Bot 私聊窗口输入：

```
/store
```

Bot 会回复「📦 存储模式已开启」，进入收集状态。

> ⏱ 10 分钟无操作自动关闭

---

### 第二步：发送资源

在会话期间，**发送的所有内容都会被收集**。支持以下方式：

#### ① 直接发送文件

直接往聊天窗口里拖入或发送：图片、视频、文件/文档、音频

#### ② 发送相册

一次性选择多张图片/视频发送（Telegram 原生相册），Bot 自动识别并整组存入。

#### ③ 转发消息

从任意频道/群组/聊天中转发消息到 Bot：
- 单条转发 → 直接存入
- 转发相册 → 整组存入
- 从存储频道转发 → 不会重复存储，只记录引用

#### ④ 发送 TG 链接

直接发送 Telegram 消息链接，Bot 在完成打包时统一处理。

**范围链接**——一次引用多条消息：
```
https://t.me/频道名/100-110
```
> 等同于引用 100~110 共 11 条消息

**一条消息里可以混合多个链接和文字。**

> 链接在点「完成打包」之前可以随时**编辑**修改

#### ⑤ 发送纯文本

直接发文字也会被收集（如说明、备注等），完成打包时存入。

---

### 链接处理规则（决策树）

完成打包时，Bot 按以下优先级处理：

| 优先级 | 条件 | 处理 |
|--------|------|------|
| 🥇 1 | 范围链接（100-110） | 视觉媒体组相册，非视觉/带说明的独立 |
| 🥈 2 | 单条视觉媒体 + 无说明文字 | 多个凑成相册（8个一组） |
| 🥉 3 | 单条视觉媒体 + 有说明文字 | 独立消息，保留原始说明 |
| 🥉 3 | 非视觉媒体（文件/音频等） | 独立消息 |

**DB 频道链接 vs 外部链接：**
- DB 频道链接 → 只记录 message_id，不重复存储
- 外部频道链接 → copy 到 DB 后记录新 message_id

**非 TG 链接 + 纯文字** → 合并为一条文本消息，保持超链接可点击

---

### 第三步：完成打包

点击 **「✅ 完成打包」** 按钮：
1. 处理所有暂存的文本和链接（显示实时进度）
2. 生成**分享链接**和**提货口令**

---

## 👤 用户使用

1. 点击管理员分享的链接
2. 如果设置了强制关注频道，先关注频道
3. Bot 自动发送资源包中的所有文件
4. 文件会在设定时间后**自动销毁**（默认 10 分钟）

---

## ⚙️ 配置说明

在 `.env` 文件或环境变量中配置：

| 变量 | 说明 | 默认值 |
|------|------|--------|
| `TG_BOT_TOKEN` | Bot Token（@BotFather 获取） | — |
| `APP_ID` | Telegram API ID | — |
| `API_HASH` | Telegram API Hash | — |
| `CHANNEL_ID` | 存储频道 ID | — |
| `OWNER_ID` | 机器人所有者 ID | — |
| `ADMINS` | 管理员 ID（空格分隔） | — |
| `FORCE_SUB_CHANNEL` | 强制关注频道 ID | — |
| `AUTO_DELETE_TIME` | 文件自动销毁时间（秒），0 = 不删除 | `600` |
| `PROTECT_CONTENT` | 防止用户转发文件 | `True` |
| `STORE_SESSION_TIMEOUT` | 存储会话无操作超时（秒） | `600` |
| `STORE_ALBUM_WAIT` | 相册合并等待时间（秒） | `0.5` |

## 💡 常见问题

**Q: 链接发错了可以改吗？**
A: 可以，直接编辑已发送的消息，Bot 自动更新。

**Q: 支持私有频道链接吗？**
A: 支持，格式为 `https://t.me/c/频道ID/消息ID`，Bot 需要是该频道成员。

**Q: 一个资源包最多放多少个文件？**
A: 没有硬性限制，建议控制在合理范围内。
